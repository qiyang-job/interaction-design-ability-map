# Interaction Design Ability Map

一个实用型的交互设计问题判别工具，帮助把日常工作里的零散优化翻译成可复盘、可表达、可沉淀的交互设计能力。

在线访问：

https://qiyang-job.github.io/interaction-design-ability-map/

## 内容结构

- 我的真实问题
- 先问用户卡在哪里
- 六类底层设计能力
- 易混概念边界
- 复盘表达生成器
- 可交互判别器
- 项目复盘表达
- 使用方法
- 产品化方向

## 核心定位

这个项目不做交互设计百科，而是做一个更窄、更实用的“交互设计复盘翻译器”：

> 输入一次具体的小优化，输出它背后的用户卡点、底层能力、设计动作和复盘表达。

## 适合场景

- 写项目复盘、周报、述职材料
- 准备作品集或面试案例
- 判断一个体验问题属于流程、信息、布局、操作、状态还是规则
- 把“我改了按钮 / 页面 / 提示”翻译成更专业的设计能力表达

## 设计原则

- 目录支持展开和收起，阅读时可以把空间让给正文
- 使用固定字号层级和 4pt 间距体系，避免页面节奏松散或层级混乱
- 配色保持黑白灰中性系统，只用深浅灰建立层级，不使用暖色背景
- 使用 Lucide 开源线性图标作为导航和理解提示，避免装饰性图形干扰阅读
- 容器表达以轻量分隔线、表格关系和局部灰底为主，避免左侧粗描边式强调
- 信息展示先聚合再展开：每个模块先给记忆点或判断结论，再展示表现、动作和复盘表达
- 表格内容需要像方法工作表，而不是传统数据表，减少硬白底和密集网格带来的阅读压力
- 移动端优先保证可读性和目录可控，不隐藏核心功能
